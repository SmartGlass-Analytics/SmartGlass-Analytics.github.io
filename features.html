Core Features

Bluetooth Pairing and Connection to Glasses

Purpose: Allow the user to pair and connect the Elo Android tablet to Brilliant Labs XYZ smart glasses using Bluetooth.

How to Use:

  • Open the SmartGlass app on the tablet.
  • Tap "Scan for Devices" on the Bluetooth screen.
  • Select your XYZ glasses from the device list.
  • Tap "Connect" to initiate pairing.

Developer Details:
Core Code: BluetoothService.kt
Built on Android BluetoothAdapter and BluetoothGatt APIs.

Sending Data to Glasses

Purpose: After establishing a Bluetooth connection, the app sends simple text data (e.g., a "Welcome" message or sample stats) to the glasses display.

How to Use:

  • Once connected, go to the Send Message screen.
  • Enter a short text message into the field.
  • Tap "Send" to transmit the message to the glasses.

Developer Details:
Core Code: MessageSender.kt
Uses Bluetooth GATT services to send short BLE packets (under 100 characters).

Reconnection and Disconnection Management

Purpose: Handle cases where the Bluetooth connection drops or the glasses power off unexpectedly.

How to Use:

  • If the connection drops, the app attempts an automatic reconnection.
  • Users can manually Disconnect and Reconnect through the app settings.

Developer Details:
Auto-reconnect and manual disconnect are handled inside BluetoothService.kt.

Developer Testing and Logging

Purpose: Enable developers to test Bluetooth functionality, simulate sending messages, and debug connection issues.

How Developers Use It:

  • Use Android logcat output to view Bluetooth connection status.
  • Simulate sending short sample messages without full API integration.
  • Test maximum payload limits through trial messages.

Developer Details:
Debug tags and simulated tests are available in BluetoothService and MessageSender classes.

Future and Planned Features

Purpose: Add functionality to pull and display lineup statistics from Synergy data or example datasets.

Current Status: Partial backend Python code exists (apipractice.py) for fetching lineup data. However, there is currently no Android UI button or Kotlin code to trigger lineup stats fetching inside the app yet.

Planned Action:

  • Add a "View Lineup Stats" button to the Android application interface.
  • Connect the button to display lineup data on the tablet and/or glasses.
